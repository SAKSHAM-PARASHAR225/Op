Deadlock — Numerical Problems

Min/Max resource formula problems

DL-1From PPT 1.3.1 — Slide 13★★★ Classic Exam▼
A system has 4 processes (P1, P2, P3, P4) and a single type of resource. The maximum needs of each process are: x1=3, x2=4, x3=5, x4=2.
Find: (a) Maximum units that ensure deadlock   (b) Minimum units that ensure NO deadlock.
Formula: Max ensuring deadlock = Σxi - n. Min for no deadlock = (Σxi - n) + 1

📋 Complete Solution

Given: n = 4 processes x1 = 3, x2 = 4, x3 = 5, x4 = 2 Step 1: Calculate Σxi (sum of all maximum needs) Σxi = 3 + 4 + 5 + 2 = 14 Step 2: Maximum units ensuring DEADLOCK = Σxi - n = 14 - 4 = 10 WHY? In worst case, each process holds (xi-1) units: P1 holds 2, P2 holds 3, P3 holds 4, P4 holds 1 Total held = 2+3+4+1 = 10 Each waits for 1 more unit → Circular wait → DEADLOCK! Step 3: Minimum units ensuring NO DEADLOCK = (Σxi - n) + 1 = 10 + 1 = 11 WHY? With 11 units, even in worst case, one process gets its 1 extra unit, completes, releases all its resources → chain reaction → no deadlock!
✅ (a) Max ensuring Deadlock = 10   |   (b) Min ensuring NO Deadlock = 11
DL-2Variation5 marks▼
A system has 3 processes with maximum resource needs: P1=4, P2=3, P3=5. Find the minimum number of resource units that guarantees the system is always deadlock-free.
Same formula. n=3, find Σxi first.

📋 Solution

n = 3 processes x1 = 4, x2 = 3, x3 = 5 Σxi = 4 + 3 + 5 = 12 Max ensuring deadlock = 12 - 3 = 9 Min ensuring no deadlock = 9 + 1 = 10 Verification (worst case): P1 holds 3 (needs 1 more), P2 holds 2 (needs 1 more), P3 holds 4 (needs 1 more) Total held = 9. Each needs 1 more. If only 9 units exist → all wait → DEADLOCK With 10 units: one process gets its needed unit, completes, releases → SAFE
✅ Minimum units for no deadlock = 10

Address Space Calculations

AS-1Physical Address Bits2 marks▼
The physical address space of a system contains 32K words. How many bits are needed for a physical address?

📋 Solution

Physical address space = 32K words = 32 × 1024 = 32768 words = 2^15 words Physical address size = log₂(2^15) = 15 bits
✅ 15 bits needed for physical address
AS-2Address Space Size2 marks▼
The physical address of a system uses 16 bits. Word size = 4 bytes. What is the physical address space size in bytes?

📋 Solution

Address space (in words) = 2^16 words (since 16-bit address) Word size = 4 bytes Address space in bytes = 2^16 × 4 = 2^16 × 2^2 = 2^18 bytes = 256 KB
✅ Physical address space = 2^18 bytes = 256 KB
AS-3Logical Address Bits2 marks▼
A process has a logical address space of 64K words. Physical address space = 32K words. How many bits are needed for: (a) logical address (b) physical address?

📋 Solution

Logical address space = 64K = 2^16 words → Logical address size = 16 bits Physical address space = 32K = 2^15 words → Physical address size = 15 bits
✅ Logical = 16 bits. Physical = 15 bits. (Logical > Physical because virtual memory extends address space)

PG-2Address Bits for Paging3 marks▼
Logical address space = 2^32 bytes. Page size = 2^12 bytes (4KB).
Find: (a) Number of pages in logical address space (b) Bits for page number (c) Bits for offset

📋 Solution

Logical address space = 2^32 bytes → total address bits = 32 Page size = 2^12 bytes → offset needs 12 bits (a) Number of pages = 2^32 / 2^12 = 2^20 = 1,048,576 pages (~1 million) (b) Bits for page number = 32 - 12 = 20 bits (c) Bits for offset = log₂(page size) = log₂(2^12) = 12 bits Logical address structure: [20 bits page number | 12 bits offset] Note: Page table has 2^20 entries × 4 bytes = 4MB per process!
✅ Pages: 2^20. Page number bits: 20. Offset bits: 12.

DS-7Rotational Latency Calculation2 marks▼
A disk rotates at 7200 RPM. Seek time = 10ms. Transfer time = 0.5ms. Calculate: (a) Rotational latency (b) Total disk access time.

📋 Solution

Rotational speed = 7200 RPM (rotations per minute) Time for one full rotation: = 60,000 ms / 7200 RPM = 8.33 ms per rotation Average rotational latency = (1/2) × time per rotation = (1/2) × 8.33 ms = 4.17 ms Total disk access time = Seek + Rotational Latency + Transfer = 10 + 4.17 + 0.5 = 14.67 ms
✅ Rotational latency ≈ 4.17ms. Total access time ≈ 14.67ms
